Данный проект — это многостраничный веб-сайт-фанат, посвященный творчеству студии Ghibli. Сайт служит интерактивной галереей фильмов, позволяя пользователям знакомиться с произведениями Хаяо Миядзаки и Исао Такахаты, а также фильтровать их по жанрам.
Проект создан как пример моей работы для портфолио frontend-разработчика.

Посмотреть получившийся результат: https://alan81.github.io/Free_Topic_Studio_Ghibli/

Какие проблемы решает этот проект:
1. Удобство: все в одном месте, не нужно искать по страницам.
2. Структурированное отображение данных: Проект превращает неструктурированную информацию о фильмах Ghibli в понятную карточку с постером, описанием и жанрами.
3. Пример современной Frontend-архитектуры: Для работодателя этот проект демонстрирует умение создавать SPA-подобное приложение с маршрутизацией без перезагрузки страницы.
4. Визуальная эстетика: Сайт решает проблему «скучных» информационных порталов, предлагая дизайн, вдохновленный атмосферой самой студии Ghibli.

Технологии:
1. React (с использованием .jsx синтаксиса)
2. React Router DOM (useNavigate, useParams для навигации и динамических маршрутов)
3. React Hooks (useState, useEffect для фильтрации и управления состоянием)
4. JavaScript (ES6+)
5. CSS3 (адаптивная верстка, кастомная стилизация)
6. GitHub Pages (хостинг и деплой)

Практическая польза проекта:
Для пользователей:
1. Быстрый поиск фильма по жанру.
2. Глубокое погружение в детали выбранного тайтла через отдельную страницу (благодаря useParams).
3. Плавная навигация между страницами без потери контекста.
Для моего развития:
1. Закрепила на практике работу с динамическими маршрутами (/movie/:id).
2. Отработала логику фильтрации массива данных (жанры) с сохранением состояния.
3. Освоил деплой React-приложения на GitHub Pages.

Как можно развить проект:
1. Добавить поисковую строку (по названию или режиссеру) — усложнит логику фильтрации.
2. Создать личный кабинет, чтобы велась статистика просмотренно/добавлено в изображенное
3. Реализовать бэкенд (например, на Node.js + Express или Firebase):
   - Хранить список фильмов в базе данных.
   - Добавить возможность ставить оценки/лайки.
   - Сделать форму для добавления отзывов пользователей.
4. Добавить «темную тему» — улучшит UX и покажет навык работы с контекстом React.
5. Интегрировать API (например, открытое Ghibli API) для автоматического получения данных.
6. Написать unit-тесты (Jest + React Testing Library) — для демонстрации ответственности.
7. Оптимизировать производительность (lazy loading изображений, React.memo).
8. Адаптировать под телефон, для просмотра с мобильного устройства.

Особенности реализации
1. Маршрутизация: Приложение многостраничное — переход между списком фильмов и детальной страницей происходит без перезагрузки через <Link> и useNavigate().
2. Динамические страницы: Страница каждого фильма строится на основе id из URL, полученного через useParams().
3. Фильтрация: Реализован компонент с кнопками жанров, при клике на который обновляется отображаемый список (локальный state).
